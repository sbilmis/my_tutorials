How to Log in to a Server via SSH Without Needing to Enter a Password

To set up passwordless SSH login, follow these steps to generate an SSH key pair and configure your server to recognize it. This allows you to connect to your server without entering a password every time.

Step 1: Generate an SSH Key Pair on Your Local Machine

  • Use the command ssh-keygen -t rsa -b 4096{.verbatim} to generate a new SSH key pair.
  • When prompted, choose the location to save the key. The default path is ~/.ssh/id_rsa{.verbatim}.
  • You may opt to enter a passphrase to secure the key or leave it empty for simpler access.

Step 2: Copy the Public Key to Your Server

  • To enable passwordless login, copy your public key to the server using: ssh-copy-id your_username@server_address{.verbatim}
  • Replace your_username{.verbatim} with your server's username and server_address{.verbatim} with the server\'s IP or domain.

Step 3: Log In Without a Password

  • After setting up, log in to the server without a password using: ssh your_username@server_address{.verbatim}

Step 4: Troubleshooting

  • *Permissions*: Ensure that the ~/.ssh{.verbatim} directory has 700{.verbatim} permissions and ~/.ssh/authorized_keys{.verbatim} has 600{.verbatim} permissions on the server.
  • *Server Configuration*: Verify that PubkeyAuthentication{.verbatim} is set to yes{.verbatim} in the SSH configuration file on the server (/etc/ssh/sshd_config{.verbatim}).

Step 5: Using an SSH Agent for Managing Passphrases

  • If your SSH key has a passphrase, you can use an SSH agent to avoid repeated passphrase entry.

  • Start the SSH agent:

shell eval $(ssh-agent -s) ssh-add ~/.ssh/id_rsa

Common SSH Commands

  • Here is a list of useful commands for managing SSH keys and connections:

``` shell # Generate a new SSH key if it does not exist cd ~/.ssh ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

# Copy the public key to the server ssh-copy-id remote_username@server_ip_address ## Once the user is authenticated, the public key will be appended to the remote user authorized_keys file and connection will be closed.

# If ssh-copy-id is unavailable, use this alternative method: cat ~/.ssh/id_rsa.pub | ssh remote_username@server_ip_address "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys" ```

  • Additional resources: Passwordless SSH Login Guide

Changing the SSH Key Password

If you need to change the passphrase of your SSH key for added security or convenience, follow these steps to update it.

  • To change the passphrase of your SSH key, use:

shell ssh-keygen -p -P oldpasswd -N newpassword -f ~/.ssh/id_rsa

  • Alternatively, run interactively:

shell ssh-keygen -p -f ~/.ssh/id_rsa

  • When prompted, choose a passphrase with at least four characters, which can include words, numbers, and punctuation for stronger security.

Using SSH Agent to Manage Passphrase

If you used a passphrase when creating your SSH key and want to avoid entering it every time, you can use an SSH agent to manage your keys.

  • Start the SSH agent and add your key:

shell eval $(ssh-agent -s) ssh-add ~/.ssh/id_rsa

  • This enables secure, convenient server access without needing to enter the passphrase every time.

Automating SSH Key Setup for Multiple Servers Using sshpass

When deploying to multiple servers, `sshpass` can automate SSH key setup by bypassing the password entry process for each server.

  • Use the following command:

shell sshpass -p 'your_password_here' ssh-copy-id user@server

  • Clear sensitive commands from history with history -d "line_number".

  • For more automation, use `expect` to script interactions:

``` expect #!/usr/bin/expect

set timeout 20 set password "your_password_here" set server "server_address" set user "your_username"

spawn ssh-copy-id $user@$server expect "password:" send "$password\r" expect eof ```

Keeping SSH Sessions Alive

If your SSH session disconnects after inactivity, you can configure keep-alive settings to maintain the connection.

  • To apply keep-alive settings globally for all hosts:

``` shell # Edit SSH config file (~/.ssh/config) chmod 600 ~/.ssh/config

Host remotehost HostName remotehost.com ServerAliveInterval 240

# To apply keep-alive settings globally for all hosts, use: Host * ServerAliveInterval 240 ```
