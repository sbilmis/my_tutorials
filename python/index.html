Virtual Environments

A Python virtual environment is an isolated environment where you can install specific versions of Python and Python packages without affecting the global (system) Python installation. This is useful for:

  • Avoiding conflicts between different projects.
  • Ensuring reproducibility (e.g., using specific versions of libraries like numpy).
  • Managing dependencies cleanly.

Common Tools for Managing Virtual Environements

  1. venv (built in)

    • Comes with python 3.3+
    • simple and lightweight
    • to create a virtual environment
    python3 -m venv myenv

  2. virtualenv

    • More feature-rich than venv

    • Works with older python versions

    • to set up:

      pip install virtualenv virtualenv myenv

  3. conda

    • Popular in data science

    • Manages both Python and non-Python packages

    • conda create -n myenv python=3.9

  4. pyenv

    • manages multiple Python versions

    • can create virtual environments using pyenv-virtualenv

    • pyenv install 3.9.6 pyenv virtualenv 3.9.6 myenv

    • In this tutorial, we will focus on pyenv

Pyenv {#pyenv-1}

  • To install pyenv and pyenv-virtualenv
## install pyenv and pyenv-virtualenv
brew install pyenv
brew install pyenv-virtualenv

  • Add the following to .bash_profile

    export PYENV_ROOT="$HOME/.pyenv" export PATH="$PYENV_ROOT/bin:$PATH" eval "$(pyenv init --path)" eval "$(pyenv init -)" eval "$(pyenv virtualenv-init -)"

  • Note that these lines should be added after exporting PATH for homebrew in bash_profile

    export PATH="/opt/homebrew/sbin:$PATH"
export PATH=/opt/homebrew/bin:$PATH

  • Useful commands about pyenv

    ## List all Python versions available to pyenv pyenv versions

    ## To list the available Python versions with pyenv pyenv install -l

    ## To install python 3.9.21 pyenv install 3.9.21

    ## when installed; you can see the files in ~/.pyenv/versions ## run pyenv versions to check pyenv versions

    ## to list existing virtual environments pyenv virtualenvs

  • Let us create a virtual environment for the examples and exercises in the book https://a.co/d/eusJ8zC;

## create a folder and change directory to that
mkdir raschka_book
cd raschka_book

## create a virtual enviroenment name raschka_book with python 3.9.21
pyenv virtualenv 3.9.21 raschka_book


## list the existing virtualenvs
pyenv viryaualenvs

## Note that you may see two results for the same environemnt. That is accepted. One of them is just a symlink

## to activate the virtualenv
pyenv activate raschka_book

## once activated you will see it on the terminal.  to check which python version is activated
python --version

## note that you will get different result if you run this command outside this folder.

  • Create a requirements.txt

    numpy==1.21.2 scipy==1.7.0 scikit-learn==1.0 matplotlib==3.4.3 pandas==1.3.2

  • Then run

    pip install -r requirements.txt

  • To verify the installation; to see the installed packages:

    pip list

  • If you add or remove dependencies in the future, you can update the requirements.txt file. To generate an updated requirements.txt file based on the currently installed packages, run:

    pip freeze > requirements.txt

  • How to delete a pyenv virtual environment?

pyenv virtualenv-delete <venv-name>

Troubleshooting:

Replace <venv-name>{.verbatim} with the name of the virtual environment you want to delete.
